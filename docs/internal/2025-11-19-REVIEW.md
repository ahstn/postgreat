# PostGreat Code Review (2025-11-19)

## Executive Summary
- Core layering (CLI in `src/main.rs`, config parsing, checker orchestration, reporter) is clean and keeps `anyhow` at the binary boundary with `snafu` elsewhere, matching AGENTS.md.
- Several business-critical heuristics diverge from the documented playbooks: shared buffer sizing is currently broken, WAL guidance omits `min_wal_size`/bursty I/O checks, and planner/index modules skip SSD/soft-delete/BRIN logic from docs/1 and docs/2.
- Test coverage does not guard the documented formulas; regressions (e.g., shared_buffers math) slipped through and no table/index heuristic tests exist.

## Critical Issues
- `shared_buffers` recommendation is effectively `0MB` because the computation divides the already-megabyte value by `1024^2`; this suppresses the 25% of RAM rule and the 8GB cap from docs/1. Example: 16GB spec yields `0`, so no alert is emitted. (src/analysis/memory.rs:31–37)
- WAL analysis never inspects `min_wal_size` and cannot flag “bursty I/O” per docs/1; only `max_wal_size` is tiered, so defaults of 1GB/80MB can persist unnoticed. (src/analysis/wal.rs:10–63)
- Planner cost tuning unconditionally targets SSD/NVMe values without detecting storage type; doc/1 calls for setting `random_page_cost = 1.1` when SSD/NVMe is present. On spinning disks this will under-cost random I/O, and on unknown storage we lack an explicit gate. (src/analysis/planner.rs:20–55)
- Index maintenance logic omits the key playbook items from docs/2: no detection of soft-delete partial index opportunities, no BRIN suggestions for time-series tables, and no guidance to prefer `CREATE INDEX CONCURRENTLY` for HA builds. Current analysis only surfaces unused, low-selectivity, and failed index-only cases. (src/analysis/table_index/indexes.rs:48–66)
- Workload detection for checkpoint timing always returns “OLTP” when stats are absent (`is_oltp_workload` defaults to `true`), so OLAP guidance (15–30 min checkpoint_timeout) is never recommended for low-connection environments. (src/analysis/wal.rs:148–167)

## Refactoring Recommendations
- Fix shared buffer sizing and cap: compute 25% directly in MB and cap at 8192MB to match docs/1; avoid division by `1024.0 * 1024.0`.
  - Before: `let recommended_mb = (spec.memory_mb() as f64 * 0.25 / (1024.0 * 1024.0)) as u64;` (src/analysis/memory.rs:31)
  - After:
    ```rust
    let recommended_mb = (spec.memory_mb() as f64 * 0.25).round() as u64;
    let recommended_mb = recommended_mb.min(8 * 1024);
    ```
- Add storage-aware planner gating: thread a `StorageKind` (Hdd|Ssd) into `SystemStats` or derive it from `effective_io_concurrency`/cloud hints, and only emit the 1.1 recommendation when storage is SSD/NVMe; otherwise keep 4.0–2.0 guidance explicit.
  - Before: unconditional `random_page_cost` suggestion to 1.1 (src/analysis/planner.rs:20–55)
  - After (sketch):
    ```rust
    if stats.storage_kind == StorageKind::Ssd && current > 1.1 {
        suggest("random_page_cost", "1.1", ...)
    } else if stats.storage_kind == StorageKind::Hdd && current > 4.0 {
        suggest("random_page_cost", "4.0", ...)
    }
    ```
- Extend WAL checks to cover `min_wal_size` and size-triggered checkpoints: parse `min_wal_size`, warn when <1–2GB, and flag cases where `max_wal_size` <= current write volume (e.g., via `pg_stat_bgwriter.checkpoints_req` rising faster than `checkpoints_timed`).
- Implement docs/2 indexing heuristics: query for tables with soft-delete predicates (columns named `is_deleted`/`deleted_at`) lacking matching partial indexes, surface BRIN candidates for large, append-only tables (`relkind='r'`, `n_live_tup` high, monotonic `created_at`), and prefer `CREATE INDEX CONCURRENTLY` in emitted rationale for online builds.

## Missing Test Cases
- Unit tests for `shared_buffers` sizing with 16/64/256 GB specs to assert the 25% rule and 8GB cap, preventing regressions like the current zero-value issue.
- Planner tests that verify SSD vs HDD branches for `random_page_cost`/`effective_io_concurrency` recommendations once storage gating is added.
- WAL tests covering `min_wal_size` and OLTP/OLAP branching in `checkpoint_timeout`; currently `is_oltp_workload`’s default masks the OLAP path.
- Table/index analysis tests using fixtures from `tests/_data` to assert detection of soft-delete partial index opportunities, BRIN candidates, and “create concurrently” guidance, alongside existing unused/low-selectivity cases.
